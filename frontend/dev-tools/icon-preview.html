Farbverhalten

Alle SVGs nutzen stroke="currentColor", sie erben also die Farbe von ihrem Container. Im Pool-Header zum Beispiel wird die Pool-Farbe (z.B. orange) durchgereicht. Im Datei-Listing bleibt es navy (über die existierende .doc-icon-Regel).

Einfacher Rückweg falls's anderen nicht gefällt

Beim Verdrahten lege ich einen kleinen Helper an: frontend/src/components/Icon.jsx mit zwei exportierten Komponenten <PoolIcon emoji="📚" /> und <FileTypeIcon type="pdf" />. Oben in der Datei eine einzige Konstante: 

// Set to false to revert to emoji rendering across the entire UI.
// Database keeps the emoji values regardless — switch is purely visual.
const LINE_ICONS_ENABLED = true

Auf false stellen → komplette UI rendert wieder die Emojis. Kein Datenbank-Eingriff, kein erneutes Auswählen, kein Code-Refactor. Eine Zeile. Wenn das Team zerrissen ist: pro User auch via localStorage-Toggle möglich, aber das ist erst nötig wenn jemand laut wird.

Wenn dir einer nicht gefällt

Sag mir den Namen ("Stern"/"Glühbirne"/etc.) und was dich stört (zu detailliert, zu abstrakt, falsche Form). Ich passe an, oder schlage 1-2 Alternativen vor. Wenn alle gut sind: einfach „alle gut" sagen, dann verdrahte ich sie in die Komponenten.
